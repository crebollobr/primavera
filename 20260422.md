
# Introdução e Estrutura do Linux

Bem-vindos ao mundo do Linux! Na cultura DevOps, o Linux é o sistema operacional que move a internet. Quase todos os servidores, bancos de dados e sistemas em nuvem rodam alguma versão de Linux.

Hoje, não usaremos o mouse. Vamos conversar diretamente com o servidor através de uma tela preta chamada **Terminal**, usando um protocolo seguro chamado **SSH**.

Para acessar o nosso laboratório, abra o seu terminal e digite o seguinte comando:
`ssh aluno@ssh.chr.eti.br`

## 1. Navegando pelo Sistema: A Regra do Telefone

No Linux, tudo é organizado em pastas (diretórios). O comando principal para se mover entre essas pastas é o **`cd`** (Change Directory / Mudar de Diretório). 

Para entender como se mover, pense nos caminhos do Linux como se fossem **ligações telefônicas**:

* **Caminho Relativo Simples (`cd pasta`):** É como ligar para um celular da **mesma cidade**. Você só precisa digitar o número local. O Linux entende que a pasta que você quer acessar já está dentro de onde você se encontra agora.
* **Caminho Relativo Externo (`cd ../pasta`):** É como ligar para **outra cidade**. Você precisa usar o DDD. O `..` significa "voltar uma pasta para trás" (sair da sua cidade) para então entrar em outro ramo (outra cidade).
* **Caminho Completo ou Absoluto (`cd /pasta`):** É como fazer uma **ligação internacional**. Você precisa colocar o código do país (+55), o DDD e o número completo. Ao começar com a barra `/` (que chamamos de pasta raiz), você está dando o endereço completo, absoluto e exato desde o começo do computador, não importa onde você esteja no momento. Por exemplo, o caminho completo até um arquivo seria `/home/aluno/Documentos/relatorio/arquivo.txt`. Se você estiver na pasta `/home/aluno/Downloads`, usaria o caminho relativo `../Documentos/relatorio/arquivo.txt` para chegar a este mesmo arquivo.
* **Diretório Home do Usuário (`cd` puro ou `cd ~`):** Funciona como um atalho que leva você imediatamente de volta para a sua própria pasta de usuário (sua casa), independentemente de onde você esteja no sistema.
* **Diretório Home de Outro Usuário (`cd ~login`):** Leva você diretamente para a pasta "home" do usuário específico chamado "login" (ou o nome de qualquer outro usuário após o til).

## 2. O Mapa do Linux: Principais Diretórios

A estrutura do Linux se parece com uma árvore de cabeça para baixo. Tudo começa na pasta principal chamada Raiz, representada apenas pelo símbolo `/`.

Abaixo da raiz `/`, temos pastas muito importantes:
* **/home:** Onde ficam os arquivos pessoais dos usuários comuns. (Sua pasta será `/home/aluno`).
* **/bin:** Onde ficam guardados os comandos básicos do sistema (como o próprio comando `cd`).
* **/var:** Onde ficam arquivos que crescem com o tempo, como os *logs* (registros de tudo o que acontece no servidor).
* **/tmp:** A pasta de arquivos temporários. Sempre que o computador reinicia, ela é apagada.

---

## 3. O Cérebro do Servidor: A pasta `/etc`

Atenção especial aqui! O diretório **`/etc`** é onde o Linux guarda todas as **configurações do sistema**. Se você alterar algo aqui de forma errada, pode quebrar o servidor. 

Dentro do `/etc`, existem arquivos de texto cruciais para o funcionamento e segurança do Linux:

* **`/etc/passwd`:** É a lista telefônica do sistema. Aqui ficam registrados todos os usuários que existem no servidor e as informações básicas deles.
* **`/etc/shadow`:** O cofre forte. Aqui ficam guardadas as senhas criptografadas dos usuários. Por segurança, um usuário comum não consegue ler esse arquivo. Somente o administrador supremo (chamado de **root** ou usando o comando **sudo**) tem permissão para acessá-lo.
* **`/etc/group`:** Assim como organizamos pessoas em departamentos numa empresa, o Linux usa esse arquivo para organizar os usuários em grupos (ex: grupo de estagiários, grupo de diretores) para facilitar as permissões.
* **`/etc/os-release`:** É o "RG" do sistema. Se você ler esse arquivo, ele dirá exatamente qual é o sistema operacional (ex: Ubuntu, CentOS, Debian) e qual versão está rodando.
* **`/etc/crontab`:** É o despertador do servidor. Esse arquivo guarda as tarefas agendadas. Se quisermos que o servidor faça um backup sozinho toda madrugada às 3h, é neste arquivo que escrevemos a regra.


# Linux: Linha de Comando (Básico)

Nesta aula, daremos os primeiros passos práticos no terminal Linux. Quando não temos uma interface gráfica (mouse e janelas), precisamos de comandos de texto para nos localizar, criar arquivos, visualizar conteúdos e buscar informações.

Abaixo estão os comandos essenciais para a sua "sobrevivência" no terminal.

---

### 1. Comandos de Localização e Navegação

* **`pwd` (Print Working Directory):**
    * **O que faz:** Funciona como o GPS do Linux. Ele mostra o caminho completo (absoluto) da pasta exata onde você está no momento.
    * **Exemplo:** Se você digitar `pwd` e pressionar Enter, o terminal pode responder `/home/aluno/documentos`. Isso confirma a sua localização atual no sistema.

* **`cd` (Change Directory):**
    * **O que faz:** Usado para navegar pelo sistema, ou seja, entrar ou sair de pastas.
    * **Exemplo:** `cd /etc` leva você direto para a pasta de configurações. Se você digitar `cd ..` (com dois pontos), o sistema volta uma pasta para trás.

* **`ls` (List):**
    * **O que faz:** Lista os arquivos e as pastas que estão dentro do diretório atual.
    * **Exemplo:** Digitar apenas `ls` mostra os nomes simples dos arquivos. Se você digitar `ls -l`, o terminal mostrará uma lista detalhada, contendo os tamanhos dos arquivos, datas de modificação e permissões.

---

### 2. Comandos de Criação

* **`mkdir` (Make Directory):**
    * **O que faz:** Cria uma nova pasta (diretório) vazia no local onde você está.
    * **Exemplo:** O comando `mkdir projetos` vai criar uma pasta chamada "projetos".

* **`touch`:**
    * **O que faz:** Cria um arquivo de texto totalmente vazio. (Ele também serve para atualizar a data e hora de modificação de um arquivo que já existe).
    * **Exemplo:** O comando `touch relatorio.txt` cria um arquivo em branco com o nome "relatorio.txt".

---

### 3. Comandos de Visualização e Texto

* **`echo`:**
    * **O que faz:** Imprime (escreve) um texto diretamente na tela do terminal.
    * **Exemplo:** Ao digitar `echo "Olá Mundo"`, o terminal simplesmente exibirá a frase *Olá Mundo* na linha de baixo.

* **`cat` (Concatenate):**
    * **O que faz:** Lê um arquivo e joga todo o seu conteúdo de uma só vez na tela do terminal. É ideal para ler arquivos de texto pequenos.
    * **Exemplo:** O comando `cat /etc/os-release` exibirá as informações do sistema operacional de forma imediata na tela.

* **`less` e `more`:**
    * **O que fazem:** São "leitores de páginas". Diferente do `cat`, eles são usados para ler arquivos muito grandes sem inundar a sua tela de uma vez só. Eles mostram o conteúdo página por página. O `less` é uma versão mais moderna que permite rolar o texto tanto para baixo quanto para cima usando as setas do teclado.
    * **Exemplo:** `less /var/log/syslog` abrirá o arquivo gigante de registros do sistema de forma pausada para leitura. Para sair do modo de leitura do `less`, basta apertar a tecla `q` (quit).

---

### 4. Comandos de Filtro e Contagem

* **`grep` (Global Regular Expression Print):**
    * **O que faz:** Funciona como uma "lupa de pesquisa". Ele varre o conteúdo de um arquivo e exibe apenas as linhas que contêm a palavra exata que você buscou.
    * **Exemplo:** O comando `grep "root" /etc/passwd` não vai mostrar o arquivo inteiro; ele vai exibir apenas as linhas que possuem a palavra "root" escrita nelas.

* **`wc` (Word Count):**
    * **O que faz:** Conta o número de linhas, palavras e caracteres de um arquivo. A opção mais utilizada por administradores é a `-l` (lines), que conta apenas o número de linhas.
    * **Exemplo:** O comando `wc -l /etc/passwd` dirá exatamente quantas linhas o arquivo tem (o que, neste caso, indica a quantidade de usuários registrados no servidor).

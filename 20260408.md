

# 🖥️ Aula Introdutória: Desmistificando a Infraestrutura de TI

Bem-vindos! Hoje vamos entender alguns dos termos mais famosos do mundo da tecnologia e, em seguida, colocar a mão na massa para ver como um sistema virtualizado funciona na prática.

## Parte 1: A Teoria (Sem "Techês" Complicado)

Para entender esses conceitos, vamos usar uma analogia com **moradia**.

### 1. Máquina Virtual (VM)
Imagine que você tem um terreno enorme. Construir uma **Máquina Virtual (VM)** é como construir uma casa inteira, do zero, dentro desse terreno. 
* Ela precisa da sua própria fundação, encanamento, fiação (Sistema Operacional próprio, como Windows ou Linux). 
* Ela é pesada, consome muitos materiais (memória RAM e processador), mas é totalmente isolada e independente. Se a casa pegar fogo, o resto do terreno fica seguro.
* **Resumo:** É um "computador de mentira" rodando dentro do seu computador de verdade. Ele acha que é uma máquina física real.

### 2. Containers
Agora, imagine que em vez de construir várias casas isoladas, você construiu um **prédio de apartamentos**. 
* Os apartamentos (Containers) compartilham a mesma fundação, o mesmo encanamento e os mesmos elevadores do prédio (compartilham o Sistema Operacional da máquina principal).
* Por compartilharem a base, eles são muito mais **leves**, rápidos para construir (iniciar) e ocupam menos espaço que uma VM.

### 3. Docker
Se os containers são os apartamentos, o **Docker** é a **construtora e a imobiliária**. 
* É o programa que usamos para empacotar, criar, distribuir e rodar esses containers de forma fácil. Ele padroniza tudo: o apartamento que você constrói no seu computador vai ser idêntico ao que roda nos servidores da nuvem.

### 4. Kubernetes
Imagine que o seu negócio cresceu. Agora você não tem mais um prédio, você tem uma cidade inteira com milhares de prédios e milhões de apartamentos (containers). Fica impossível gerenciar tudo na mão.
* O **Kubernetes** é o **prefeito e o síndico superpoderoso** dessa cidade. 
* Ele monitora se um apartamento "caiu" (travou) e cria outro no lugar automaticamente. Ele decide qual prédio tem mais espaço vazio para colocar novos inquilinos. Ele organiza (orquestra) o caos.

## Parte 2: Laboratório Prático 🛠️

Agora vamos esquecer um pouco os containers e voltar para o conceito de **VM (Máquina Virtual)**. Vamos ligar um computador virtual dentro do nosso!

### Passo 1: Iniciar o VirtualBox
O VirtualBox é o nosso "gerenciador de terrenos". Ele é o programa que permite criar e rodar as VMs.
* **Ação:** Abra o aplicativo **VirtualBox** no seu computador.

### Passo 2: Importar o arquivo `alpine.ova`
O arquivo `.ova` é como uma casa pré-fabricada. Alguém já instalou o sistema e empacotou tudo num arquivo só. Nós só precisamos importar.
* **Ação:** No VirtualBox, vá em `Arquivo` > `Importar Appliance` (ou *Import Appliance*).
* Selecione o arquivo `alpine.ova` que foi disponibilizado e conclua a importação.

### Passo 3: Descobrir e Iniciar a VM
* **Ação:** Veja que uma nova máquina apareceu na lista do lado esquerdo do seu VirtualBox.
* Selecione ela e clique na seta verde gigante **"Iniciar"** (ou *Start*). Uma nova janela preta vai se abrir. Esse é o monitor do seu novo computador virtual.

### Passo 4: Fazer o Login
Quando a máquina terminar de carregar, ela vai pedir um usuário (`login`).
* **Ação:** Digite o usuário administrador supremo do Linux:
  ```bash
  root
  ```
* **Ação:** Ele vai pedir a senha (*Password*). **Atenção:** No Linux, quando você digita a senha, nada aparece na tela (nem asteriscos). Isso é normal! Digite a senha que o professor vai passar e aperte `Enter`.

### Passo 5: Onde nós estamos?
Vamos descobrir em qual pasta do sistema nós "caímos" após o login.
* **Ação:** Digite o comando abaixo e aperte `Enter`:
  ```bash
  pwd
  ```
  *(pwd significa "Print Working Directory", ou seja, "imprima o diretório de trabalho")*. Ele deve mostrar que você está em `/root`.

### Passo 6: Navegando no sistema
Vamos mudar de pasta e ir para o diretório de configurações do Linux, chamado `etc`.
* **Ação:** Digite:
  ```bash
  cd /etc
  ```
  *(cd significa "Change Directory").*

### Passo 7: Descobrindo o Linux
Existem várias "marcas" (distribuições) de Linux: Ubuntu, Fedora, Debian... Qual estamos usando?
* **Ação:** Vamos ler (usando o comando `cat`) o arquivo de identificação do sistema:
  ```bash
  cat os-release
  ```
* **Pergunta:** *Qual Linux estamos usando?*
* **Resposta:** Lendo a saída na tela, você verá o nome **Alpine Linux**. É uma versão de Linux extremamente leve e minúscula, muito usada justamente para criar containers!

### Passo 8: Descobrindo nosso endereço na rede
Para a máquina se comunicar com a internet, ela precisa de uma "placa de rede" e de um "endereço IP".
* **Ação:** Digite:
  ```bash
  ip add
  ```
* **Pergunta:** *Qual interface e endereço IP estamos usando?*
* **Resposta:** Procure na tela pela interface chamada **`eth0`** (sua placa de rede principal) ou **`enp0s3`**. Logo abaixo dela, você verá algo como `inet 10.0.2.15` ou `192.168.x.x`. Esse é o endereço IP da sua VM.

### Passo 9: Testando a conexão com a Internet
Vamos fazer um teste enviando 4 "bolinhas de ping-pong" para os servidores do Google e ver se eles respondem.
* **Ação 1 (Testando por IP):** ```bash
  ping -c 4 8.8.8.8
  ```
  *(Isso envia 4 pacotes para o servidor DNS do Google. Se você ver tempos de resposta em milissegundos, a rede está funcionando!)*

* **Ação 2 (Testando por Nome):**
  ```bash
  ping -c 4 www.google.com
  ```
  *(Isso testa se o seu Linux consegue converter um nome de site em um endereço IP corretamente).*

---

### 🎉 Conclusão
Se os `pings` responderam com sucesso: **A rede foi testada e o Linux está 100% funcional!** Você acabou de importar, acessar, operar e validar um servidor Linux virtualizado do zero.



# Laboratório Prático: O Encanador do Linux

Neste laboratório, vamos aprender que o terminal não é apenas um lugar para digitar comandos isolados, mas um sistema de "tubulações" onde a saída de um programa alimenta a entrada de outro.

## 1. As Peças do Sistema (Variáveis)
O Linux guarda informações importantes em "etiquetas" chamadas variáveis de ambiente. Vamos ver quem é você no sistema.

* **Comando:** `echo` (serve para imprimir texto na tela).
* **Ação:** Digite os comandos abaixo e observe o que aparece:

| Variável | Comando | O que ela guarda? |
| :--- | :--- | :--- |
| **$USER** | `echo $USER` | O nome do usuário logado no momento. |
| **$HOME** | `echo $HOME` | O caminho para a sua pasta pessoal. |
| **$PATH** | `echo $PATH` | A lista de pastas onde o Linux procura os programas para executar. |

## 2. Fluxos e Redirecionamentos (O encanamento básico)
Por padrão, tudo o que um comando faz sai na tela. Podemos desviar esse fluxo para arquivos usando as "setas".

### A Seta que Apaga (`>`)
O símbolo `>` joga o resultado para um arquivo, mas **cuidado**: ele apaga tudo o que existia no arquivo antes.
```bash
echo "Meu nome é $USER" > info.txt
cat info.txt
```

### A Seta que Soma (`>>`)
O símbolo `>>` adiciona o texto ao final do arquivo, sem apagar o que já estava lá.
```bash
echo "Estou aprendendo Linux" >> info.txt
cat info.txt
```
> **Dica de Segurança:** No dia a dia, prefira usar `>>` a menos que tenha certeza de que quer deletar o conteúdo anterior.

---

## 3. O Poder do Pipe (`|`)
O **Pipe** (barra vertical) é o coração do Linux intermediário. Ele pega a saída de um comando e a "empurra" diretamente para dentro de outro comando.

**Desafio:** Vamos listar todos os arquivos do diretório `/etc`, mas queremos apenas contar quantos arquivos existem lá que possuem a palavra "conf".

1. **ls /etc** (lista os arquivos)
2. **grep "conf"** (filtra apenas os que têm "conf")
3. **wc -l** (conta as linhas)

**Comando Completo:**
```bash
ls /etc | grep "conf" | wc -l
```

---

## 4. O Pulo do Gato: Redirecionando Erros (`2>&1`)
Às vezes, um comando dá erro. No Linux, a "Saída Padrão" (texto comum) e a "Saída de Erro" viajam por tubos diferentes. 

* `1` é o tubo do sucesso (stdout).
* `2` é o tubo do erro (stderr).

Se você tentar listar uma pasta que não existe e salvar em um arquivo, o erro vai aparecer na sua tela e o arquivo ficará vazio:
```bash
ls /pasta_que_nao_existe > erro.txt
```

**A Mágica:** Para capturar o erro dentro do arquivo, usamos o `2>&1` (que diz: "jogue o fluxo 2 para onde o fluxo 1 estiver indo").
```bash
ls /pasta_que_nao_existe > log_total.txt 2>&1
cat log_total.txt
```
Isso é o que separa quem sabe apenas o básico de quem sabe administrar um servidor de verdade.



# 🐧 Aula de Linux: Do Zero ao Script

### Primeiros Passos: Criando e Lendo Arquivos

No Linux, usamos o terminal para quase tudo. Para criar um arquivo de texto simples, usamos o editor **nano**. Para ler o conteúdo desse arquivo sem precisar abrir um editor, usamos o comando **cat**.

* **Exemplo:**
1. Digite `nano mensagem.txt`.
2. Escreva `boa noite` dentro do arquivo.
3. Pressione `Ctrl + O` e `Enter` para salvar, e `Ctrl + X` para sair.
4. Digite `cat mensagem.txt` para ver o texto aparecer na tela.



### O que é o Bash?

O **Bash** é o interpretador de comandos do Linux. Ele é como um tradutor: você digita algo que humanos entendem (como `ls` ou `cd`) e ele traduz para o computador executar.

### Formas de usar o Bash

Podemos usar o Bash de duas formas:

1. **Linha de comando:** Você digita um comando por vez e aperta Enter (ex: `date`).
2. **Como arquivo (Script):** Você escreve uma sequência de comandos em um arquivo e o Bash executa todos de uma vez, como uma receita de bolo.

### Criando seu primeiro programa (`.sh`)

Para criar um script, abrimos um arquivo com a extensão `.sh`. Na primeira linha, usamos obrigatoriamente o **Shebang** (`#!/bin/bash`), que avisa ao sistema que aquele arquivo deve ser lido pelo Bash.

* **Exemplo:**
Crie o arquivo `cumprimentos.sh` com o comando `nano cumprimentos.sh`.
Coloque este conteúdo:
```bash
#!/bin/bash
echo "boa noite"

```



### Por que o programa não roda direto? (O Path)

Se você tentar digitar apenas `cumprimentos.sh`, o Linux dirá que o comando não foi encontrado. Isso acontece porque o sistema só procura comandos em pastas "oficiais" (chamadas de **Path**). Ele não olha automaticamente na sua pasta atual por segurança.

### Executando com o caminho relativo (`./`)

Para rodar um arquivo que está na sua pasta agora, você deve indicar o caminho. O ponto `.` representa a "pasta atual". Então, usamos `./` antes do nome.

* **Tentativa:** Digite `./cumprimentos.sh`.
* **Resultado:** Provavelmente aparecerá "Permissão negada".

### A Flag de Execução (`chmod +x`)

Arquivos novos no Linux não têm permissão para "rodar" como programas por padrão. Para transformar seu texto em um programa executável, usamos o comando `chmod +x`.

* **Exemplo:**
Digite `chmod +x cumprimentos.sh`. Agora, ao digitar `./cumprimentos.sh`, o programa funcionará e exibirá "boa noite".

### Parâmetros (Entradas do programa)

Você pode passar informações para dentro do seu programa na hora de executá-lo. O Bash organiza isso por números:

* `$1` é a primeira palavra/número que você digitar depois do nome do programa.
* `$2` é a segunda, e assim por diante.
* **Exemplo:** Se o seu script tiver `echo "Olá $1"`, e você rodar `./programa.sh Carlos`, ele dirá "Olá Carlos".

### Variáveis e Cálculos

No Bash, as variáveis guardam textos. Para criar uma, usamos `NOME="valor"`. Para usar o valor dela depois, colocamos o cifrão na frente: `$NOME`.

Para fazer cálculos matemáticos, usamos uma estrutura especial com dois parênteses: `$(( conta ))`.

* **Exemplo:**
```bash
NUMERO=10
SOMA=$((NUMERO + 5))
echo "O resultado é $SOMA"

```



### O Comando `if` (Decisão)

O `if` testa se algo é verdade. Usamos `-eq` para comparar se números são iguais e `-ne` para diferentes.

* **Exemplo:**
```bash
if [ $1 -eq 10 ]; then
    echo "Você digitou dez!"
else
    echo "Não é dez."
fi

```



### O Comando `while` (Repetição por condição)

O `while` repete um bloco de códigos **enquanto** uma condição for verdadeira.

* **Exemplo (Contador):**
```bash
CONTADOR=1
while [ $CONTADOR -le 5 ]; do
    echo "Contagem: $CONTADOR"
    CONTADOR=$((CONTADOR + 1))
done

```


*(Nota: `-le` significa "menor ou igual")*

### O Comando `for` (Repetição por lista)

O `for` percorre uma lista de itens que você já conhece.

* **Exemplo:**
```bash
for NOME in "Ana" "Beto" "Caio"; do
    echo "Bem-vindo $NOME"
done

```
